This directory contains practical workflow guides for using the QA Prompt Library in real-world testing scenarios.

### [Sprint Testing Workflow](sprint-testing-workflow.md)
**Use for**: 2-week sprint cycles

**Covers**:
- Sprint planning and test strategy
- Daily testing activities throughout sprint
- Mid-sprint review
- Regression and integration testing
- Sprint review and retrospective
- Continuous activities

**Best for**: Agile/Scrum teams

---

### [Release Testing Workflow](release-testing-workflow.md)
**Use for**: Production releases

**Covers**:
- Release planning (T-14 days)
- Test preparation and execution
- Specialized testing (performance, security, accessibility)
- Release candidate testing
- Go/No-Go decision process
- Release day procedures
- Post-release monitoring

**Best for**: Release managers and QA leads

- **Upcoming release?** ‚Üí Use Release Testing Workflow

### 2. Adapt to Your Context
- Adjust timelines to your sprint length
- Modify activities based on your team size
- Add/remove steps as needed
- Integrate with your tools

### 3. Follow the Links
Each workflow links to relevant prompts in the library:
- Click links to access specific prompts
- Use prompts to generate test cases, code, or strategies
- Customize prompts for your needs

### 4. Track Progress
Use the checklists in each workflow to track completion:
- [ ] Mark items as you complete them
- [ ] Review daily/weekly progress
- [ ] Identify blockers early

## üí° Workflow Tips

### For Daily Workflow
- **Bookmark** frequently used prompts
- **Time-box** exploratory testing sessions
- **Batch** similar activities together
- **Review** test results every afternoon

### For Sprint Workflow
- **Plan** testing at sprint start
- **Track** progress mid-sprint
- **Communicate** blockers early
- **Reflect** in retrospective

### For Release Workflow
- **Start early** (T-14 days)
- **Define** quality gates upfront
- **Monitor** metrics continuously
- **Prepare** rollback plan
