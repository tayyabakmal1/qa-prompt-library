# Automation QA Prompts

Welcome to the Automation QA section of the QA Prompt Library. This comprehensive collection contains prompts for test automation across web, API, mobile, performance, security, and more.

## üìö Categories

### 1. Web Automation
Automate web application testing with popular frameworks.

**Frameworks Covered**:
- **Selenium** - Industry-standard web automation
- **Playwright** - Modern, cross-browser automation
- **Cypress** - JavaScript-based E2E testing
- **Puppeteer** - Chrome DevTools Protocol automation

**When to use**: Web UI automation, browser testing, E2E scenarios

### 2. API Automation
Automate API testing and validation.

**Tools Covered**:
- [Postman](api-automation/postman-prompts.md) - API collection creation and testing
- **REST Assured** - Java-based REST API testing
- **Newman** - CLI execution of Postman collections
- **Karate DSL** - BDD-style API testing

**When to use**: API testing, service validation, integration testing

### 3. Mobile Automation
Automate mobile application testing.

**Frameworks Covered**:
- **Appium** - Cross-platform mobile automation
- **XCUITest** - iOS native automation
- **Espresso** - Android native automation

**When to use**: Mobile app testing, cross-platform testing

### 4. Framework Design
Design robust, scalable test automation frameworks.

**Topics Covered**:
- Framework architecture
- Page Object Model (POM)
- Design patterns
- Best practices

**When to use**: Building new frameworks, refactoring existing code

### 5. CI/CD Integration
Integrate tests into continuous integration pipelines.

**Platforms Covered**:
- Jenkins
- GitHub Actions
- GitLab CI
- Azure DevOps

**When to use**: Setting up CI/CD, automating test execution

### 6. Performance Testing
Load and performance testing automation.

**Tools Covered**:
- **JMeter** - Java-based load testing
- **Gatling** - Scala-based performance testing
- **K6** - JavaScript performance testing
- **Locust** - Python load testing

**When to use**: Load testing, stress testing, performance benchmarking

### 7. Contract Testing
Consumer-driven contract testing.

**Frameworks Covered**:
- **Pact** - Consumer-driven contracts
- **Spring Cloud Contract** - Spring-based contract testing

**When to use**: Microservices testing, API contracts

### 8. Security Testing
Automated security testing and vulnerability scanning.

**Topics Covered**:
- OWASP Top 10 testing
- Penetration testing
- Vulnerability assessment

**When to use**: Security validation, compliance testing

### 9. Accessibility Testing
Automated accessibility compliance testing.

**Standards Covered**:
- WCAG 2.1/2.2 compliance
- Screen reader testing
- Keyboard navigation

**When to use**: Accessibility compliance, inclusive design validation

### 10. Database Testing
Database validation and testing.

**Types Covered**:
- SQL database testing
- NoSQL database testing
- Data migration testing

**When to use**: Data validation, migration testing

### 11. Cloud Testing
Cloud platform testing strategies.

**Platforms Covered**:
- AWS testing
- Azure testing
- Google Cloud Platform testing

**When to use**: Cloud service validation, infrastructure testing

### 12. Test Reporting
Test execution reporting and analytics.

**Tools Covered**:
- Extent Reports
- Allure Reports
- Custom reporting solutions

**When to use**: Test result visualization, stakeholder reporting

### 13. Parallel Execution
Parallel and distributed test execution.

**Approaches Covered**:
- TestNG parallel execution
- Selenium Grid
- Cloud-based execution (BrowserStack, Sauce Labs)

**When to use**: Reducing execution time, scaling tests
